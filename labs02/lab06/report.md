## Теоретическая часть

**Грамотное программирование (Literate Programming)** - методология, предложенная Дональдом Кнутом, где код и документация объединены в единый документ.

**R Markdown** - технология для создания динамических документов, объединяющих код R, результаты вычислений и текстовое описание.

**Основные компоненты:**
- YAML заголовок - метаданные документа
- Markdown текст - форматированный текст
- Code chunks - блоки кода R
- Inline code - встроенный код в тексте

### Задание 1: Динамический документ R Markdown

#### Структура документа

Документ включает:
- YAML заголовок с настройками вывода

#### Ключевые фрагменты кода

```r
# Создание тестовых данных
set.seed(123)
random_data <- rnorm(100, mean = 50, sd = 10)

# Вычисление статистических характеристик
mean_value <- mean(random_data)
median_value <- median(random_data)
sd_value <- sd(random_data)
```

#### Inline код в документе

Среднее значение данных составляет `r round(mean_value, 2)`, медиана равна `r round(median_value, 2)`.

#### Визуализация

## Ответы на контрольные вопросы

### 1. Грамотное программирование (Literate Programming)

Методология, где код и документация объединены в единый документ. Улучшает читаемость и поддерживаемость кода.

### 2. R Markdown

Технология для создания динамических документов, объединяющих код R, результаты и текст. Поддерживает различные форматы вывода (HTML, PDF, Word).

### 3. Структура R Markdown документа

- YAML заголовок с метаданными
- Markdown текст с форматированием
- Code chunks для выполнения кода
- Inline code для вставки результатов

### 4. CSS стилизация в R Markdown

CSS позволяет настраивать внешний вид HTML документа. Файл CSS указывается в YAML заголовке и применяется при компиляции.
