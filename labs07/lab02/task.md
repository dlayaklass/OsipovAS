### **Лабораторная работа (часть 2): Тестирование асинхронного кода в Go**

**Цель работы:** Освоить методики тестирования конкурентного кода, горутин, каналов и асинхронных операций в Go. Научиться писать надежные тесты для параллельных программ.

**Стек технологий:**
*   **Язык программирования:** Go 1.19+
*   **Операционная система:** Ubuntu 20.04/22.04 LTS
*   **Инструменты:** Go testing framework, `go test`, race detector
*   **Библиотеки:** `testing`, `sync`, `time`, `context`

---

### **Задание на практическую реализацию:**

#### **Часть 1: Тестирование базовых горутин и каналов**

**1.1. Тестирование простых горутин**
*   **Задача:** Написать тесты для Counter и ProcessItems.
*   **Требования:** Проверка конкурентного доступа, отсутствие гонок данных.
*   **Демонстрация:** Корректность работы с WaitGroup.

**1.2. Тестирование каналов**
*   **Задача:** Написать тесты для работы с каналами.
*   **Требования:** Проверка таймаутов, буферизованных каналов.
*   **Демонстрация:** Корректность обработки данных через каналы.

#### **Часть 2: Тестирование Worker Pool**

**2.1. Базовое тестирование**
*   **Задача:** Написать тесты для Worker Pool.
*   **Требования:** Проверка распределения задач, обработки результатов.
*   **Демонстрация:** Корректность работы пула воркеров.

**2.2. Конкурентное тестирование**
*   **Задача:** Тестирование конкурентной отправки задач.
*   **Требования:** Проверка отсутствия гонок данных.
*   **Демонстрация:** Устойчивость к нагрузке.

#### **Часть 3: Тестирование HTTP сервера**

**3.1. Базовое тестирование**
*   **Задача:** Написать тесты для HTTP обработчиков.
*   **Требования:** Проверка корректности ответов.
*   **Демонстрация:** Работа с httptest.

**3.2. Нагрузочное тестирование**
*   **Задача:** Тестирование конкурентных запросов.
*   **Требования:** Проверка счетчика запросов.
*   **Демонстрация:** Устойчивость к нагрузке.

### **Критерии оценки:**

*   **Удовлетворительно:** Реализованы базовые unit-тесты (части 1-2)
*   **Хорошо:** Реализованы комплексные тесты с интеграционными тестами (части 1-3)
*   **Отлично:** Полное покрытие тестами + детектор гонок + покрытие кода не менее 70%
