### **Лабораторная работа №5: Применение паттернов проектирования**

**Цель работы:** Освоить практическое применение паттернов проектирования для рефакторинга и улучшения кодовой базы системы учета сотрудников из ЛР №2.

**Стек технологий:**
* **Язык программирования:** Python 3.x
* **Дополнительные библиотеки:** `abc`

---

### **Задание на практическую реализацию:**

Работа представляет собой рефакторинг системы из ЛР №4 с применением 4 паттернов проектирования.

#### **1. Singleton (Одиночка)**
* **Задача:** Создать класс `DataStorage` для хранения данных о сотрудниках.
* **Требования:** Гарантировать единственный экземпляр хранилища в рамках приложения.
* **Методы:** `get_instance()`, `add_employee()`, `get_employee()`, `get_all_employees()`
* **Демонстрация:** Показать, что многократное создание объекта возвращает один и тот же экземпляр.

#### **2. Builder (Строитель)**
* **Задача:** Создать `EmployeeBuilder` для пошагового создания сложных объектов сотрудников.
* **Требования:** Реализовать fluent-интерфейс (цепочка вызовов). Возможность создания сотрудников разных типов с различными параметрами.
* **Методы:** `set_id()`, `set_name()`, `set_department()`, `set_base_salary()`, `set_type()`, `build()`
* **Демонстрация:** Построение объектов Employee, Manager, Developer, Salesperson с разными конфигурациями.

#### **3. Strategy (Стратегия)**
* **Задача:** Реализовать различные стратегии расчета бонусов.
* **Требования:** Создать интерфейс `BonusStrategy` с реализациями:
  - `PerformanceBonus` - бонус как процент от зарплаты
  - `SeniorityBonus` - бонус за стаж работы
  - `ProjectBonus` - фиксированный проектный бонус
* **Демонстрация:** Динамическая смена стратегий расчета для одного сотрудника.

#### **4. Observer (Наблюдатель)**
* **Задача:** Реализовать систему уведомлений об изменениях в системе.
* **Требования:** Создать `NotificationSystem` с методами подписки/отписки. Уведомлять об изменениях зарплат сотрудников.
* **Методы:** `add_observer()`, `remove_observer()`, `update()`
* **Демонстрация:** Работа системы уведомлений при изменении зарплаты.

---

### **Требования к демонстрации:**

* Показать взаимодействие паттернов в комбинированном примере
* Продемонстрировать преимущества после рефакторинга

### **Требования к отчету:**
1. **Код:** Исходные тексты всех классов с паттернами
2. **Диаграммы:** UML-диаграмма классов

4. **Анализ:** Преимущества и недостатки каждого паттерна
