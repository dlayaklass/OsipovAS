## Теоретическая часть

**Git** - распределенная система контроля версий, позволяющая отслеживать изменения в коде и организовывать командную работу.
**GitHub** - веб-сервис для хостинга проектов.

**Стратегии ветвления:**
*   **Git Flow:** Сложная модель с ветками `main`, `develop`, `feature/*`, `release/*`, `hotfix/*`.
*   **GitHub Flow:** Упрощенная модель с веткой `main` и feature-ветками.

**Pre-commit hooks:** Скрипты для автоматической проверки качества кода перед коммитом.

### Часть 2. Настройка pre-commit

Был создан файл конфигурации `.pre-commit-config.yaml` для автоматической проверки кода.

```yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.4.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files
  - repo: https://github.com/psf/black
    rev: 23.3.0
    hooks:
      - id: black
        language_version: python3
```
