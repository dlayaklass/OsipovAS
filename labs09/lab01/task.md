# Лабораторная работа №9
## Принципы поддержания качества кода

**Тема:** Рефакторинг системы учета сотрудников на основе принципов чистого кода и SOLID.

**Цель работы:** Освоить практические навыки рефакторинга существующего кода, применения принципов SOLID, устранения "запахов кода" и улучшения архитектуры системы.

### Стек технологий:
- Python 3.x
- pylint, black, mypy
- Git

### Задание:

1. **Анализ кода** - запустить статический анализ с помощью pylint
2. **Выявить "запахи кода"** - длинные методы, большие классы, дублирование
3. **Применить SRP** - выделить валидацию, сериализацию, логирование
4. **Применить OCP** - паттерн Strategy для расчета зарплат
5. **Применить ISP** - разделить интерфейсы
6. **Применить DIP** - внедрить dependency injection
7. **Устранить дублирование** - применить DRY, KISS, YAGNI

### Критерии оценки:
- Удовлетворительно: рефакторинг 2-3 классов с 2+ принципами SOLID
- Хорошо: рефакторинг основных модулей, 4+ принципа SOLID
- Отлично: полный рефакторинг с применением всех принципов SOLID
